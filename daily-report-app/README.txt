========================================
  日报与周报管理工具 v1.3
  使用手册
========================================

【简介】
一款纯本地的日报与周报管理工具。基于 Node.js 运行时，零外部
依赖，数据全部存储在本地磁盘。按时间段记录每日工作，自动统计
日报、生成周报，支持通过 Server酱/企业微信机器人将日报推送
到手机。

【功能一览】
  日报填写  - 按时间段记录工作，自动计算用时
  日报统计  - 按日期汇总各类工作时长与进度
  周报统计  - 按周展示各分类每天用时及总计
  文字周报  - 自动生成可复制的周报文本
  推送设置  - 配置 Server酱 或企微机器人，定时推送到微信
  定时任务  - 配合 Windows 任务计划程序实现每日自动推送

========================================
  所需环境
========================================

1. Node.js 18+
   下载地址: https://nodejs.org/
2. curl.exe（Windows 10/11 自带，用于定时任务推送）

========================================
  快速开始
========================================

1. 启动服务器
   双击 start.bat，或运行:
   node server.js

   服务器默认监听 http://localhost:3456

2. 打开浏览器访问
   http://localhost:3456

3. 数据存储目录（自动创建）
   D:\ReportData\

4. （可选）添加开机自启
   双击 add-startup.ps1 → "使用 PowerShell 运行"
   之后每次登录时服务器自动启动

========================================
  使用指南 - 日报填写
========================================

操作入口: 顶部标签栏「日报填写」

日期选择
  - 使用 ← → 按钮切换日期，或点击日期输入框直接选择
  - 「今天」按钮快速返回当日

添加记录
  - 开始/结束时间: 选择时间段，用时自动计算（精确到分钟）
  - 工作项目: 输入项目名，支持自动补全已有项目
  - 工作分类: 下拉选择已有分类，或输入新名称后点 + 号添加
  - 项目概括: 关联同类工作的摘要标签，支持自动补全
  - 进展状况: 已完成 / 开展中 / 部分完成 / 未完成 / 暂停 / 取消
  - 工作内容: 描述具体工作事项
  - 备注/问题: 记录遇到的问题或备注

编辑记录
  - 点击已有记录列表中的任意条目，自动回填到表单
  - 修改后点击「修改记录」保存，或点击「取消」放弃
  - 每条记录右侧有删除按钮

========================================
  使用指南 - 日报统计
========================================

操作入口: 顶部标签栏「日报统计」

  - 选择日期，自动汇总当日各分类工作数据
  - 展示内容包括: 各分类用时、工作条目列表、进展状况
  - 底部显示当日总用时

========================================
  使用指南 - 周报统计
========================================

操作入口: 顶部标签栏「周报统计」

  - 默认显示上周数据，可切换「本周」或「上一周」
  - 使用 ← → 按钮切换周次
  - 按工作分类展示周一至周六每天的用时（列联表）
  - 每行末尾显示该分类本周总计
  - 底部显示每天合计和本周总用时

========================================
  使用指南 - 文字周报
========================================

操作入口: 顶部标签栏「文字周报」

  - 按周次自动生成纯文本格式的工作总结
  - 格式: 序号. 项目概括 - 进展状况 - 用时(h)
  - 末尾汇总总用时
  - 点击「复制」按钮一键复制到剪贴板
  - 适用于粘贴到邮件、企业微信、钉钉等场景

========================================
  推送设置
========================================

操作入口: 顶部标签栏「设置」

本工具支持两种推送通道，用于将日报定时推送到微信:

Server酱（推荐）
  1. 打开 https://sct.ftqq.com 用微信扫码登录
  2. 复制你的 SendKey（形如 SCT123456...）
  3. 在设置页选择 Server酱，粘贴 SendKey
  4. 勾选「启用每日自动推送」，保存
  5. 点击「发送测试」验证

企业微信机器人
  1. 在企业微信群中添加群机器人，复制 Webhook URL
  2. 在设置页选择企业微信机器人，粘贴 URL
  3. 勾选启用，保存并测试

========================================
  定时任务（自动推送）
========================================

推送配置保存并启用后，需要安装 Windows 定时任务:

  右键 install-tasks.ps1 → 使用 PowerShell 运行

创建的任务:
  - DailyReportPush  每天 08:30  推送昨日日报
  - WeeklyReportPush 每周一 09:00 推送上周周报

也可手动测试推送（无需管理员权限）:
  curl.exe http://localhost:3456/api/push/daily

========================================
  文件结构
========================================

daily-report-app/
  server.js              后端服务器（零依赖，纯 Node.js 内置模块）
  start.bat              一键启动 + 打开浏览器
  start_server.cmd       仅启动服务器（不打开浏览器）
  package.json           项目元信息
  README.txt             本文件
  add-startup.ps1        添加开机自启脚本
  install-tasks.ps1      安装定时推送任务脚本
  scripts/
    push-daily.ps1       日报推送脚本（供定时任务调用）
    push-weekly.ps1      周报推送脚本（供定时任务调用）
  public/
    index.html           主页面
    css/style.css        macOS 风格样式
    js/app.js            核心交互逻辑
    js/settings.js       推送设置逻辑

数据存储（D:\ReportData\）:
  YYYY-MM-DD.json        每天的工作记录数据
  _categories.json       工作分类列表
  _projects.json         项目列表
  _push_config.json      推送通道配置

========================================
  注意事项
========================================

  - 本工具完全本地运行，数据不经过任何外部服务器
  - 服务器端口为 3456，如需修改请编辑 server.js 顶部 PORT 变量
  - 数据文件为 JSON 格式，可直接用文本编辑器查看和备份
  - 如需备份数据，备份 D:\ReportData 目录即可
  - 如果 3456 端口被占用，修改 PORT 后重启服务器
